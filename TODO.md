## 第二层：策略深度

### 2.1 压力/腐蚀系统（核心代价机制） ✅

**设计目标：** 让"持续性交换"有累积代价，防止无脑最优解。

- [x] 定义压力值（暂称"墨渍/噪音"）的数据结构
  - 全局数值，玩家可见
  - 区分"当场结算交换"（不涨压力）和"持续性交换"（涨压力）
  - 持续性交换 = 改自己武器形容词、改环境词（+2压力）
- [x] 压力阈值与后果
  - 阈值 5：句子中随机出现一个不可交换的"噪音词"占位
  - 阈值 10：某个已有词块的值被替换为"扭曲的"
  - 阈值 15：某个词块槽位被永久锁定（本次 run 内）
- [x] 压力的视觉表现
  - 顶部进度条显示墨渍值
  - 游戏背景随压力变暗变灰
  - 横幅提示压力事件
- [x] 压力的消除途径
  - 事件节点：净化泉水（-3）、净化卷轴（-5）
  - 休息节点恢复 HP（间接减少压力需求）

### 2.2 地图选路系统 ✅

**设计目标：** 让战斗间有"路线规划"层面的策略，创造"普通关攒资源、Boss 关爆发"的节奏。

- [x] 地图结构设计
  - 3 层 + Boss 层，每层 2-3 个可选节点
  - 最后一层固定为 Boss（巨龙）
- [x] 节点类型
  - 普通战斗：低风险低奖励
  - 精英战斗：高风险高奖励（HP+、伤害+）
  - 事件：剧情选择（净化泉水、流浪商人、古老石碑）
  - 休息：恢复 8HP
  - 商店（待定）
- [x] 地图 UI
  - 弹出式地图面板，节点按层排列
  - 节点图标+颜色区分类型
  - 已走过的路径标记为 visited
- [x] 地图与压力的交互
  - 事件节点提供净化机会
  - 路线选择 = 风险-收益权衡

### 2.3 敌人意图系统 ✅

**设计目标：** 让每回合的交换决策是"针对性回应"而不是"找全局最优解"。

- [x] 敌人意图类型
  - 攻击（显示伤害数字）
  - 防御（本回合减伤）
  - 锁定（锁定玩家某个词块槽位）
  - 强化（增加自身攻击力）
  - 净化（清除场上负面效果，无可净化则轻击）
- [x] 意图显示 UI
  - 敌人状态栏旁显示意图图标+数值+颜色
  - 每回合开始时 roll 新意图，基于权重随机
- [x] 意图对交换决策的影响
  - 防御时攻击无效→省次数做别的交换
  - 锁定时→抢先把好词换走
  - 强化时→抢先换走敌人好词

### 2.4 数值重新平衡 ✅

- [x] 拉大战斗间的难度梯度
  - 第1层：HP×0.8, 伤害×0.8, 5次交换（弱敌，不交换也能过）
  - 第2层：标准数值, 4次交换（需要精准交换）
  - 第3层：HP×1.3, 伤害×1.2, 4次交换（精英为主）
  - Boss层：巨龙 HP35 伤害10, 6次交换（需要组合策略）
- [x] "主语交换"机制重新设计
  - 采用方案 B+C：消耗 2 次交换次数 + 涨 5 点压力
  - 仍然保留反转效果，但代价极高，不再是无脑翻盘
- [x] HP 交换限制
  - 采用方案 B：HP 交换涨 3 点压力
  - HP 仍然可以交换，但有明确代价

### 2.5 三模式操控系统（交换/添加/删除） ✅

**设计目标：** 将"对文字的操控"从单一交换扩展为三种操作，统一消耗"笔力"资源，用压力作为代价区分器。

- [x] 统一资源"笔力"（替代原"交换次数"）
  - 所有操作共享同一资源池
  - UI 全局替换为"笔力"
- [x] 交换模式（原有功能）
  - 拖拽交换同类型词块，消耗 1 笔力（主语交换 2 笔力 + 5 墨渍）
- [x] 删除模式
  - 点击词块抹除，消耗 2 笔力 + 4 墨渍
  - 被删除的位置变为空槽位（可用添加模式填充）
  - 关键词块（HP、名字、笔力）不可删除
- [x] 添加模式
  - 从词块库选取词块，点击空槽位插入，消耗 1 笔力
  - 类型必须匹配（形容词槽位只能放形容词）
- [x] 词块库系统
  - 击败敌人后战利品词块自动加入词块库
  - 词块库面板展示所有收集的词块
  - 选中词块后自动切换到添加模式
- [x] 模式切换 UI
  - 三个模式按钮（交换/添加/删除）
  - 删除模式下词块显示十字准星光标
  - 添加模式下空槽位脉冲发光提示

## 设计悬而未决的问题

1. **"持续性交换"如何精确定义？** 改自己武器=持续性？改敌人的属性=当场结算？改环境=哪种？
2. **压力值衰减还是只增不减？** 如果只增不减，净化点就是唯一出路；如果自然衰减，节奏会更宽松。
3. **地图的层数和分支数**——太多选择会让单局时间过长，太少则路线规划没意义。
4. **一个 run 的目标时长是多少？** 这决定了地图深度和战斗场次。
