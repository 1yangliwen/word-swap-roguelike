# Word Swap Roguelike — 核心设计文档

> 版本：v0.1（初始概念）  
> 最后更新：2025年5月  
> 一句话概括：一切屏幕上的文字和数字都可以互相交换位置，交换后游戏状态立即按新的语义结算。

---

## 一、核心体验目标

玩家在一段段叙述性文字中寻找最优的"交换"——用有限的交换次数，通过互换数字、形容词、主语等文字元素，扭转战斗局势、触发意外连锁反应、发现隐藏组合。

设计关键词：快速决策（10秒/回合）、资源博弈（交换次数跨战斗继承）、解锁惊喜（特殊组合触发连锁）。

---

## 二、核心循环

### 2.1 单回合流程

```
[生成战斗描述] → [玩家阅读] → [选择是否交换（0~N次）] → [按新语义结算] → [下一回合]
```

1. 系统生成当前回合的描述文字（敌人行动 + 环境描述 + 结算预告）
2. 玩家阅读所有文字，识别可交换的词块（高亮显示）
3. 玩家选择两个词块互换位置，或选择不操作（保留交换次数）
4. 系统按交换后的新文字语义结算效果（伤害、治疗、状态变化等）
5. 进入下一回合

### 2.2 单局流程

```
[普通战斗 x3~4] → [精英战斗] → [普通战斗 x3~4] → [Boss战斗] → 下一层
```

- 普通战斗：文字量少，压力低，主要目的是让玩家"攒"交换次数
- 精英战斗：出现稀有词块和特殊组合机会，诱惑玩家消耗交换次数
- Boss战斗：文字量暴涨，句子多、可交换对象多，是消耗资源的高潮时刻

### 2.3 战斗间的衔接：总结语系统

每场战斗结束后，系统根据**这场战斗的实际过程**动态生成一句总结语。总结语是下一场战斗中的常驻可交换素材池。

**总结语提取规则（从本场战斗中提取）：**

| 提取要素 | 决定总结语中的… | 示例 |
|----------|----------------|------|
| 最后一击的方式 | 动词 + 名词 | "斩杀""刺穿""焚烧" |
| 玩家受到的最大伤害来源 | 形容词 | "炽热的""剧毒的" |
| 场上残留的环境状态 | 环境词 | "燃烧的空气""冰封的地面" |
| 一个关键数字 | 数字词块 | 剩余血量/造成总伤害/最大单次伤害 |

**示例——同一场战斗的不同打法产生不同总结语：**

打法A（用燃烧的剑击杀，场地着火，剩12血）：
> "你用**燃烧的**剑斩杀了**哥布林**，**炽热的**空气中弥漫着**12点**余烬"

打法B（交换主语让哥布林自己打死自己）：
> "**哥布林**在**混乱中**刺穿了自己，留下了**生锈的**匕首和**5点**血迹"

**设计意义：** 玩家当前战斗的决策不仅影响本场胜负，还决定了下一场的可交换素材池。这形成跨战斗的策略链：

```
当前战斗决策 → 战斗过程 → 总结语内容 → 下一场素材池 → 下一场决策空间
```

**屏幕结构（每场战斗开始时）：**

```
┌─────────────────────────────────┐
│ [总结语] 上一场留下的素材池      │  ← 常驻，词块可交换
│─────────────────────────────────│
│ [环境描述] 当前场景              │  ← 常驻，词块可交换
│─────────────────────────────────│
│ [战斗描述] 本回合的行动          │  ← 每回合更新
│─────────────────────────────────│
│ [玩家状态栏] ❤️ HP ⚔️ 装备       │  ← 常驻，数字/形容词可交换
│ [敌人状态栏] 💀 HP               │  ← 常驻，数字可交换
└─────────────────────────────────┘
```

## 三、交换系统

### 3.1 交换次数（核心资源）

- 每局开始时获得初始交换次数（暂定3次）
- 普通战斗通关奖励：+1次交换
- 交换次数**跨战斗继承**，不会自动回复
- 策略核心：普通关尽量不花，攒到Boss关爆发

### 3.2 交换规则

| 规则 | 说明 |
|------|------|
| 同类型交换 | 数字只能和数字换，形容词只能和形容词换，主语只能和主语换 |
| 全局可换 | 当前屏幕上所有可见的词块都可以互相交换（包括UI元素上的数字） |
| 立即结算 | 交换后新的句子立刻生效，不可撤回 |
| 单次操作 | 一次交换 = 选中两个同类型词块互换位置 |

### 3.3 可交换词块类型

| 类型 | 示例 | 交换效果 |
|------|------|----------|
| 数字 | 8点、12点、3层 | 数值直接互换 |
| 形容词 | 生锈的、锋利的、燃烧的 | 改变关联对象的属性数值 + 可能触发组合 |
| 主语 | 你、哥布林、巨龙 | 改变动作的施受关系 |
| 名词 | 剑、盾、草地 | 改变交互对象，可能触发组合 |

### 3.4 UI穿透

以下UI元素上的数字/词也是可交换对象：

- 玩家血条上的生命值数字
- Boss血条上的生命值数字
- 回合计数器上的数字
- 玩家装备栏里的形容词（如"锋利的"铁剑）

交换UI元素上的数字和战斗描述中的数字后，UI立即更新反映变化。

## 四、词块属性系统

### 4.1 形容词数值表（部分示例）

每个形容词有一个基础数值修正，作用于它所修饰的名词：

| 形容词 | 攻击修正 | 防御修正 | 特殊标签 |
|--------|----------|----------|----------|
| 生锈的 | -3 | -2 | 无 |
| 锋利的 | +5 | 0 | 无 |
| 燃烧的 | +3 | -1 | 火焰 |
| 冰冻的 | +2 | +2 | 冰霜 |
| 诅咒的 | +7 | -4 | 暗影 |
| 神圣的 | +4 | +4 | 光明 |

### 4.2 特殊组合触发（连锁反应）

当特定标签的形容词与特定名词组合时，触发额外效果：

| 形容词标签 | 名词 | 触发效果 |
|------------|------|----------|
| 火焰 | 草地/森林 | 场地燃烧：每回合所有人受2点伤害 |
| 冰霜 | 河流/水面 | 冻结通道：敌人下回合无法行动 |
| 暗影 | 光明（词块碰撞） | 湮灭：双方词块消失，触发爆炸伤害 |
| 火焰 + 冰霜 | 同一句子中 | 蒸汽迷雾：所有词块随机打乱一次 |

玩家初始不知道这些组合，通过游玩逐步发现。发现后记录在"词典"中（局外永久解锁的收集系统）。

### 4.3 交换合法性原则

核心约束：**任何形容词都能修饰任何名词，不存在"不合法"的交换。** 只有效果好坏之分，没有语法错误。

交换发生时，两个词块各自离开原位、进入对方的位置，两句话同时改变。设计上需要保证：

1. 每个形容词对每个名词都有定义好的效果（哪怕只是"无特殊效果，数值归零"）
2. 每个名词放在任何句子的宾语/环境位上都语法合理（通过模板句式保证）
3. 玩家做交换前能预览两句话交换后的样子（但不显示具体数值结果，只显示文字变化）

示例："燃烧的"从"剑"移动到"草地"上 → 剑失去火焰加成变成无词条，草地触发场地燃烧。原本修饰草地的"干燥的"移动到剑上 → "干燥的剑"= 无属性加成，合法但弱。

## 五、战斗示例（纸面原型用）

### 场景：第一层·第一场普通战斗

**初始状态：**
- 玩家生命值：20
- 哥布林生命值：15
- 玩家交换次数：3

**回合1 — 屏幕文字：**

> 战斗描述："**哥布林**用**生锈的**匕首对**你**造成了**5点**伤害"  
> 环境描述："你站在**干燥的** **草地**上"  
> 玩家状态栏：[❤️ **20**] [⚔️ **锋利的**铁剑]  
> 哥布林血条：[💀 **15**]

**可交换词块：**
- 数字：5、20、15
- 形容词：生锈的、干燥的、锋利的
- 主语：哥布林、你
- 名词：匕首、草地、铁剑

**玩家可能的决策：**

| 操作 | 效果 | 代价 |
|------|------|------|
| 不交换 | 老实吃5点伤害，血量20→15 | 0次交换 |
| 交换"5"和"15" | 哥布林对你造成15点伤害，但哥布林血量变成5 | 1次交换 |
| 交换"哥布林"和"你" | 变成"你用生锈的匕首对哥布林造成5点伤害" | 1次交换 |
| 交换"生锈的"和"锋利的" | 哥布林武器变锋利（伤害5+8=13），你的剑变生锈 | 1次交换（亏，不该换） |

**最优决策分析：**

如果这只是第一场普通战斗，可能最好的选择是"不交换"——吃5点伤害无所谓，把交换次数留给后面。除非玩家发现"干燥的"+"草地"在后续关卡可能有用，想提前把某个形容词换过来布局。

## 六、技能系统

### 6.1 设计定位

交换（笔力）是跨多场战斗积攒的稀缺"大招"——改写现实的终极能力。技能则是**常规回合的基本战斗手段**，填充交换冷却期间的战斗空白。两者分工明确：

| 系统 | 频率 | 定位 |
|------|------|------|
| 技能 | 每回合可用 | 日常战斗手段 |
| 交换（笔力） | 跨多场积攒 | 翻盘大招 |
| 收藏品（遗物） | 被动生效 | 改变交换规则与风格 |

三层节奏嵌套：**小技能（每回合）→ 大技能（攒几回合）→ 交换（攒几场战斗）**。

### 6.2 技能槽与资源

- **4个技能槽**：固定可见，不存在抽牌随机性，每回合都可选择释放
- **法力值**：跨回合继承（不像卡牌游戏每回合清空），每回合自动回复 2-3 点
- **小技能**消耗 2-3 法力（每回合放一个的节奏）
- **大技能**消耗 5-7 法力（需要攒 2-3 回合）
- 法力够的情况下，一回合可以连续释放多个技能

跨回合继承的设计意义：玩家需要规划"这回合全花掉放两个小技能，还是攒一回合放大技能"。和交换的"攒资源爆发"逻辑一致，形成节奏感。

### 6.3 技能与词块的关系

**部分技能（主要是输出类）释放时会在战斗叙事中产生新词块，这些词块直接参与当回合或后续回合的战斗结算。** 产生词块不是所有技能的强制特性，而是部分技能的额外效果。

- 输出类：大部分会产生进攻性词块（因为攻击本身会影响战斗描述）
- 防御类：部分会产生防护性词块
- 操控类：基本不产生词块（价值在于改变局势本身）

技能产生的词块有即时结算效果（如持续伤害、减伤），交换能进一步利用它们是锦上添花，但不是必须的。

### 6.4 技能分类

#### 输出类

核心作用：造成伤害。产生的进攻性词块附着在敌人描述上，影响后续结算。不同输出技能产生不同属性词块，可能触发现有的 combo 连锁系统。

代表技能：

| 技能名 | 法力消耗 | 效果 | 产生词块 |
|--------|----------|------|----------|
| 劈砍 | 3 | 造成 4 点伤害 | 无 |
| 火焰斩 | 3 | 造成 3 点伤害，下回合再扣 2 点 | "燃烧的"（附着敌人） |
| 雷霆一击 | 6 | 造成 8 点伤害，敌人下回合无法行动 | "麻痹的"（附着敌人） |

#### 防御类

核心作用：减少即将受到的伤害。虽然交换也能改变伤害数值，但交换太珍贵不值得用在日常防御上。技能提供常规的"挡一下"。

代表技能：

| 技能名 | 法力消耗 | 效果 | 产生词块 |
|--------|----------|------|----------|
| 格挡 | 2 | 本回合减少 3 点伤害 | 无 |
| 石肤 | 3 | 本回合减少 4 点伤害，下回合再减 1 点 | "坚固的"（附着玩家） |
| 反击姿态 | 5 | 本回合减少 4 点伤害，将减免量反弹给敌人 | 无 |

#### 操控类

核心作用：改变局势，不直接处理伤害。为后续回合或交换创造更好的条件。

代表技能：

| 技能名 | 法力消耗 | 效果 | 产生词块 |
|--------|----------|------|----------|
| 威吓 | 2 | 敌人下回合意图强制变为防御（跳过攻击） | 无 |
| 解锁 | 3 | 解除一个被敌人锁定的词块，恢复可交换状态 | 无 |
| 暴露弱点 | 5 | 标记敌人，接下来 2 回合受到所有伤害翻倍 | 无 |

### 6.5 技能获取与替换

- **开局携带**：1-2 个初始技能（具体取决于角色/难度设定）
- **获取途径**：击败精英敌人、Boss、事件/商店奖励
- **槽位已满时**：必须替换掉一个现有技能（不可超过 4 个）

### 6.6 与压力系统的关系

技能系统**不影响压力值**。压力的来源仍然只有交换操作（主语反转、HP交换等高代价交换），净化途径仍然只有事件节点的稀有机会。技能是"安全的"日常操作，交换才是"有代价的"终极手段。

### 6.7 回合结构（更新）

```
[回合开始] → [显示敌人意图] → [玩家行动阶段：释放技能（法力够可连放）+ 可选择使用交换] → [结算] → [回合结束]
```

### 6.8 待细化

- 完整技能池需等内容系统（形容词/元素/combo 规则）扩充后批量设计
- 技能升级/强化机制（是否存在？通过什么途径？）
- 初始技能的选择是否构成"职业"差异

## 七、成长系统

### 7.1 局内成长

| 来源 | 获得 |
|------|------|
| 通过普通战斗 | +1笔力 |
| 通过精英战斗 | +1笔力 + 新技能选择 |
| 通过Boss战斗 | 进入下一层 + 新词块进入词池 |

### 7.2 收藏品/遗物系统

被动生效的收藏品，获取后持续生效直到本次 run 结束。与技能系统分工明确——技能管日常战斗操作，收藏品提供被动加成和build方向。

**获取途径：** 击败精英/Boss、事件奖励、商店购买

**稀有度梯度：**

| 稀有度 | 影响方向 | 示例方向 |
|--------|----------|----------|
| 普通 | 生存/经济 | 最大HP+5、战利品多一个词块选择、商店打折 |
| 稀有 | 技能增强 | 法力每回合多回复1点、某类技能消耗-1、技能产生的词块多持续1回合 |
| 传说 | 改变交换规则 | （待后期从实际体验中发现痛点后设计） |

**设计原则：**

- 普通和稀有收藏品数量多，单个不强但累积定义这局的强度曲线
- 传说级极少，直接改变交换的运作方式，是最强力的build定义物
- 传说级需等核心战斗循环（技能+敌人+交换）跑通后，从实际游玩体验中发现"交换在哪些场景下差一点"再针对性设计

**待细化：** 收藏品的具体条目、收藏品之间的 synergy 关系、一局最多持有多少个

### 7.3 局外成长（永久解锁）

| 系统 | 说明 |
|------|------|
| 词典 | 记录已发现的特殊组合，提供提示（但不直接给答案） |
| 词库扩展 | 通关特定条件后，新的形容词/名词进入随机池 |
| 起始天赋 | 局外解锁不同的起始加成（如初始+1笔力、起始装备词条更好等） |

## 八、敌人设计

### 8.1 设计原则

参考杀戮尖塔的敌人设计哲学：每个敌人教会玩家一件事，逼迫玩家用特定方式应对。敌人的差异化体现在三个维度：

- **机制差异**：不同敌人有独特的战斗规则（免疫、反弹、词块吞噬等）
- **节奏差异**：不同敌人改变战斗的时间压力（速攻型、蓄力型、召唤型）
- **叙事差异**：不同敌人的战斗描述提供不同数量/质量的可交换词块

### 8.2 Boss 多管血设计

Boss 采用多管血条（如 3 管），每管血量独立。设计意义：

- 防止玩家用交换直接互换血量一刀秒杀（交换只影响当前管）
- 天然形成阶段转换——每打空一管血，Boss 切换行为模式
- 给战斗节奏创造起承转合

### 8.3 第一层敌人（教学层）

教玩家基本操作，各自逼你用不同的基础能力。

| 敌人 | HP | 伤害 | 核心机制 | 教会玩家什么 |
|------|-----|------|----------|-------------|
| 哥布林 | 12 | 4 | 无特殊机制，纯沙包 | 输出技能击杀的基本节奏 |
| 毒蛇 | 8 | 2 | 每回合给玩家叠"中毒的"负面词块（回合结束扣1点） | 抢速度，不快杀就持续亏血 |
| 石壳虫 | 15 | 3 | 身上有"坚固的"词块，物理伤害减半 | 关注敌人描述里的词块；可用交换换走"坚固的"或用元素技能绕过 |

### 8.4 第二层敌人（组合层）

逼玩家做资源分配决策，需要技能配合。

| 敌人 | HP | 伤害 | 核心机制 | 逼迫的决策 |
|------|-----|------|----------|-----------|
| 盗贼 | 14 | 5 | 第一回合封印你一个技能槽（1-2回合不可用） | 先手用威吓阻止，还是忍受缺槽硬打？ |
| 巫师 | 12 | 3 | 每2回合给自己叠一个强化词块（"狂暴的""燃烧的"等），越拖越疼 | 防御苟住还是抢输出？ |
| 双子史莱姆 | 7+7 | 3+3 | 两个敌人同时行动，各自血少 | 法力分配——一回合杀一个承受另一个伤害，还是均摊输出？ |

### 8.5 第三层敌人（精英层）

需要技能+交换配合才能击败。精英是地图上的**可选节点**，玩家可以选择绕开。击败精英奖励新技能。

| 敌人 | HP | 伤害 | 核心机制 | 应对策略 |
|------|-----|------|----------|---------|
| 铁甲骑士 | 25 | 7 | 每隔一回合举盾（该回合完全免伤） | 规划输出节奏，或用"暴露弱点"在不举盾回合打爆发 |
| 咒术师 | 18 | 4 | 每回合锁定场上1-2个词块，3回合后释放大招（高伤害） | 用"解锁"保住关键词块，或在大招前用交换翻盘 |
| 噬词兽 | 20 | 5 | 描述词块极少（交换空间小），每回合吞噬你场上一个形容词词块 | 快速击杀，或用技能产生新词块补充场上素材 |

### 8.6 Boss 设计示例：巨龙（三管血）

**第一管（20HP）—— 试探期**
- 正常攻击模式，意图在攻击/防御/锁定之间切换
- 教玩家熟悉它的行为模式
- 节奏较宽松，玩家可以积攒法力准备后续阶段

**第二管（20HP）—— 狂暴期**
- 攻击力翻倍，不再使用防御意图
- 逼玩家大量使用防御技能硬抗，或用威吓跳过部分回合
- 法力压力大——防御和输出抢资源

**第三管（15HP）—— 终末期**
- 每回合锁定1-2个词块 + 大范围攻击
- 设计意图：逼玩家把积攒的交换充能用出来做最终翻盘
- 这是交换系统的"高光时刻"

### 8.7 敌人与技能的交互总结

| 敌人机制 | 对应的技能应对 |
|----------|---------------|
| 负面词块持续伤害 | 快速输出击杀 / 操控类清除 |
| 物理免疫/减伤词块 | 元素技能绕过 / 交换换走 |
| 封印技能槽 | 威吓阻止 / 忍耐等待恢复 |
| 蓄力强化 | 抢输出打断（在蓄满前击杀）|
| 锁定词块 | 解锁技能 / 交换操作 |
| 吞噬词块 | 技能产生新词块补充 / 速杀 |
| Boss多管阶段转换 | 不同阶段切换技能策略 |

### 8.8 待细化

- 每层具体出现哪些敌人的概率分布

## 九、地图系统

### 9.1 核心结构

参考杀戮尖塔的地图模型：玩家可以看到**完整的路线图**，从底部起点到顶部Boss，中间若干层节点通过连线相连。玩家只能沿连线向上走，选择决定了后续能到达哪些节点。

**可见性：** 整张地图在run开始时就完全可见（节点类型、连线关系），玩家可以提前规划路线。

### 9.2 参数（暂定，待实际游玩后调整）

- **总层数：** 10-15 层 + Boss 层（作为一个完整Act）
- **每层节点数：** 2-4 个
- **每个节点向上连接 1-3 个下一层节点**（不是全连接，形成真正的路径分支）
- **起点固定 1 个，Boss 层固定 1 个**

### 9.3 节点类型

| 节点类型 | 图标 | 内容 | 频率 |
|----------|------|------|------|
| 普通战斗 | ⚔️ | 第一/二层敌人 | 最常见 |
| 精英战斗 | 💀 | 第三层敌人，可选择绕开，奖励新技能 | 较少 |
| 事件 | ❓ | 可交换的文字事件（见第十章） | 中等 |
| 商店 | 🛒 | 购买技能/收藏品/回复品 | 较少 |
| 休息 | 🔥 | 恢复HP（或其他选择如升级技能？待定） | 较少 |
| Boss | 👑 | 多管血Boss，最终关卡 | 每Act末尾固定1个 |

### 9.4 分布规则

- **前3层不出现精英**（给玩家成型时间）
- **Boss前一层必有休息节点**（玩家可以选择走到休息补血再打Boss）
- **精英节点集中在中段**（第4-10层），且至少有一条路线可以绕开所有精英
- **事件和商店在整个地图中均匀分布**
- **第一层固定为普通战斗**（快速进入游戏节奏）

### 9.5 路线规划的策略性

地图完全可见的设计意义：

- 玩家看到前方有精英 → 当前优先去商店/事件获取准备
- 玩家看到Boss前有事件 → 留一点笔力到那里交换事件文字
- 玩家看到某条路线精英密集但奖励多 → 高风险高收益路线选择
- 玩家当前血少 → 选择通往休息节点的路线

### 9.6 待细化

- 具体层数需等战斗时长确定后调整
- 是否引入多Act结构（多个地图依次推进）
- 地图生成算法（如何保证每次run的地图有趣且可玩）
- 是否有隐藏节点/特殊路线

## 十一、交换适用场景（统一操作语言）

交换（笔力）是整个游戏的统一操作语言，贯穿所有界面。所有出现文字的场景都是潜在的交换战场，笔力在所有场景之间竞争。**笔力的全局稀缺性本身就是所有场景的天然约束，不需要额外的使用次数限制。**

### 11.1 战斗中

交换战斗描述中的词块改变回合结算。详见第三章。

### 11.2 事件中

**结构：** 事件由描述文字 + 多个选项按钮组成。描述和选项里都包含可交换词块。

**流程：** 玩家先阅读事件文字 → 可选择花笔力交换词块（改变描述和/或选项的内容）→ 选择一个选项执行 → 系统按当前文字语义结算。不交换也能正常选选项。

**示例：**

> "一位**贪婪的**商人向你展示了他的货物"
> - 选项A：花费**5枚**金币购买**浑浊的**药水
> - 选项B：花费**8枚**金币购买**锋利的**匕首
> - 选项C：离开

可能的交换操作（每次消耗1笔力）：
- 交换选项A的"5"和选项B的"8" → 变成5金币买匕首、8金币买药水
- 交换"浑浊的"和"锋利的" → 变成锋利的药水（强效）、浑浊的匕首（弱武器）
- 交换描述里的"贪婪的"和选项里的某个形容词 → 改变商人行为或物品属性

**设计意义：** 事件不只是"读文字选选项"，而是"先改题目再做选择"。交换让玩家有机会改善结果，但每一笔力都有机会成本——这里花了，战斗/收藏品/战利品就少一次。

### 11.3 战利品/收藏品获取界面

战斗胜利后的奖励描述、收藏品获取时的描述，同样包含可交换词块。

示例：
> "你获得了**生锈的**匕首和**3枚**金币"

- 把"生锈的"和身上的某个形容词交换 → 改变获得的武器品质
- 把"3"和其他可见数字交换 → 改变金币数量

**设计意义：** 玩家全程在做一个判断——"这里值不值得花笔力？"战斗、事件、奖励结算……所有场景竞争同一个稀缺资源，构成跨场景的策略决策。
